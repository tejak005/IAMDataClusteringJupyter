Cluster Core: Technical Deep Dive

Implementation Details & Methodology

🧮 Confidence Scoring: Peer Adoption Lift

Multi-factor analysis providing robust, mathematical, and explainable assessments:

Confidence / Validation Logic:
Confidence Score = (Cluster Holders / Total Cluster Population) * 100
Global Lift = Peer Adoption Rate / Global Company Adoption Rate

🎯 Peer Adoption Rate

Evaluates the exact frequency of an entitlement natively within the dynamically generated K-Means group.

📊 Global Lift

Measures how unique the access is to the peer group vs the entire enterprise (e.g. 2.0x more common locally).

⚠️ Anomaly Flags

DBSCAN assigns '-1' density markers to users who share no clear mathematical similarity with any peers.

🤖 ML Approach Overview

Clustering Algorithm

  1. Extract contextual HR features (department, job_title, location).
  2. Normalize & encode entitlements into a Binary Access Sparse Matrix.
  3. Apply dimensionality reduction via TruncatedSVD (100 components).
  4. Cluster using K-Means (Iterative Silhouette Scoring).
  5. Scan for outliers using Density-Based Spatial Clustering (DBSCAN).

Models Used

  • K-Means: Group users dynamically by access/HR patterns.
  • DBSCAN: Identify irregular groups and unique outliers.

🔧 Data Engineering

  • Contextual: One-Hot Encoded HR metadata.
  • Behavioral: Sparse aggregation of User <-> Entitlement arrays.
  • Aggregates: High-risk counts, Total access counts (StandardScaled).

Model Inference Flow

  • Export models via Joblib tracking properties.
  • FastAPI intercepts User JSON inference requests.
  • Map to matrix template, Scale, Reduce, and Cluster in real-time.
